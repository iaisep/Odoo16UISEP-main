Document Processing with AI

Descripción

Este módulo integra funciones avanzadas de inteligencia artificial para analizar, validar y gestionar flujos de documentos en Odoo. Utiliza OCR y GPT para extraer texto y evaluar documentos de manera eficiente.

Requisitos previos

  1. Odoo 16 instalado en su servidor.
  2. Dependencias del módulo (indicadas en el archivo requirements.txt):
    • Python 3.8 o superior.
    • Bibliotecas: requests, pillow, PyPDF2, pytesseract, openai.

Instalación

1. Descargar el módulo

Descargue el código del módulo y colóquelo en la carpeta addons de su instancia de Odoo.

2. Instalar dependencias

Ejecute el siguiente comando para instalar las dependencias:

pip install -r requirements.txt

Luego instala el OCR Tesseract

sudo apt install tesseract-ocr

para mas información para instalar Tesseract visite esta página:
Windows
https://ucd-dnp.github.io/ConTexto/versiones/master/instalacion/instalacion_popple_teseract_windows.html

Linux
https://ucd-dnp.github.io/ConTexto/versiones/master/instalacion/instalacion_popple_teseract_linux.html

Asegúrese de instalar el paquete de idioma Español.

3. Instalar el módulo

  • Vaya a Aplicaciones > Actualizar lista de aplicaciones.
  • Busque e instale Document Processing with AI.

4. Configurar parámetros del sistema

Acceda a Configuración > Parámetros > Parámetros del sistema y configure los siguientes valores:

  • openai_api_key: Clave API de OpenAI.
  • tesseract_route: Ruta de Tesseract en Linux (por defecto: /usr/bin/tesseract).

Uso

1. Configurar parámetros de IA

  • Vaya a Configuración > IA Personalizada > Parámetros de IA.
  • Cree un nuevo registro seleccionando:
    • Tipo de documento (Acta de grado, Notas de certificación, Registro civil).
    • Activar el motor GPT si desea usar la integración con OpenAI.
    • Configurar parámetros específicos para GPT.

2. Procesamiento de documentos

El módulo analizará automáticamente cualquier archivo subido en el modelo Adjuntos. Este análisis puede:

  • Validar el formato del documento.
  • Extraer texto utilizando OCR (Tesseract).
  • Evaluar documentos con reglas definidas o con GPT.

3. Estados del documento

Los documentos analizados serán clasificados automáticamente en los siguientes estados:

  • Aceptado (accepted): Cumple con todos los requisitos.
  • Formato incorrecto (incorrect_format): Problemas en la estructura del documento.
  • Documento incorrecto (incorrect_document): Datos inválidos o insuficientes.
  • Legibilidad deficiente (poor_readability): Legibilidad deficiente para la lectura.

4. Notificaciones automáticas

El sistema notificará automáticamente cualquier problema detectado en el procesamiento.

Consideraciones

  1. Configure correctamente la ruta de Tesseract para evitar errores en el análisis OCR.
  2. Asegúrese de tener una clave válida de OpenAI para usar la integración GPT.
  3. Los parámetros del sistema configurados manualmente no se sobrescribirán al actualizar el módulo.
